
## 一 背景+定位
Flow-GRPO 使用的训练集：
- **来源**：NQ (Natural Questions, ~80K) + DeepMath-103K (~103K)
- **总量**：182,190 个 (query, answer) 对
- **任务类型**：知识检索（NQ）+ 数学推理（DeepMath）
- **数据格式**：`{question, result, source, extra_info}`

我们的需求：
- 规模：1k-10k级别
	- ps：这个定位准不准，如果更小我怕看不出来效果
- **任务覆盖**：至少需要覆盖**搜索/检索和数学**推理两类（与原始 Flow-GRPO 训练分布一致），以支持 AgentFlow 的工具调用训练
- **质量要求**：有 ground-truth answer，可用于二进制结果奖励（GPT-4o Judge）
- **开源可用**：HuggingFace 或 GitHub 可直接下载

## 二 候选数据集
### 1 DAPO-Math-17k  

| 维度           | 详情                                                                       |
| ------------ | ------------------------------------------------------------------------ |
| **数据量**      | ~17,398 条（英文 14,116 + 中文 3,282）                                          |
| **任务类型**     | 纯数学推理（整数答案）                                                              |
| **质量评估**     | 高——ByteDance/Tsinghua 团队精心筛选，用于训练 Qwen2.5-32B 达到 AIME 2024 50 分          |
| **使用的论文**    | DAPO (ByteDance, 2025), DS-GRPO, ACE, SAI-DPO                            |
| **数据格式**     | `{prompt, solution (string)...}`                                         |
| **开源地址**     | `open-r1/DAPO-Math-17k-Processed` on HuggingFace                         |
| **与我们任务的匹配** | ✅ 数学推理——覆盖了原始训练集中 DeepMath 的功能                                           |
| **重要特点**     | 已被 DAPO 验证可在 GRPO 下有效训练；使用 batch=32, rollout=8 的配置；已被至少 23 个下游模型作为训练数据使用 |
| **数据质量**     | 高：来自 MATH 基准测试，题目经过人工审核                                                  |
| **答案准确性**    | 高——整数答案，exact match 判定                                                   |

### 2 LIMR-1.4K （数学推理 小规模高效训练）

| 维度           | 详情                                                                                |
| ------------ | --------------------------------------------------------------------------------- |
| **数据量**      | **1,389** 条（精选）                                                                   |
| **任务类型**     | 数学推理                                                                              |
| **质量评估**     | **极高**——经 Learning Impact Measurement 算法筛选，1,389 条效果 ≥ 8,523 条全量训练                |
| **使用的论文**    | LIMR: Less is More for RL Scaling (GAIR, Feb 2025)                                |
| **存储格式**     | Parquet (HuggingFace: `GAIR/LIMR`)                                                |
| **数据格式**     | `{ID，prompt,answer,source..}`                                                     |
| **核心发现**     | "A strategically selected small subset matches or beats training on 6× more data" |
| **与我们任务的匹配** | ✅ 极小的数据量，极高的训练效率——非常适合 Motivation 实验                                              |
| **重要特点**     | 证明了"少即是多"在 RL 训练中的可行性；LIM 筛选算法可作为一种数据质量保证                                         |

### 3 Natural Questions (NQ)

| 维度              | 详情                                           |
| --------------- | -------------------------------------------- |
| **数据来源**        | 真实 Google 搜索查询 + Wikipedia 页面                |
| **数据量**         | Train: **79,168** / Dev: 8,757 / Test: 3,610 |
| **存储格式**        | JSONL（FlashRAG 统一格式）                         |
| **HuggingFace** | `RUC-NLPIR/FlashRAG_datasets` (subset: `nq`) |
| **任务类型**        | 开放域问答 / 知识检索                                 |
| **数据格式**        | `{id, question，golden_answers，metadata}`     |
| **与我们任务匹配**     | 这是训练搜索/检索工具使用能力的数据--flow-grpo使用NQ作为训练主要组成部分  |
| **质量评估**        | 数据真实高/答案人工验证/难度分布是否混合类型                      |

### 4 HotpotQA
| 属性              | 详情                                                                                                                                      |
| --------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| **数据来源**        | Wikipedia-based multi-hop question answering                                                                                            |
| **数据量**         | Train: **90,447** / Validation: 7,405（distractor 配置）                                                                                    |
| **存储格式**        | Parquet                                                                                                                                 |
| **HuggingFace** | `hotpotqa/hotpot_qa`                                                                                                                    |
| **开源许可**        | CC-BY-SA-4.0                                                                                                                            |
| **任务类型**        | 多跳推理问答                                                                                                                                  |
| **数据格式**        | `{id, question，answer,type,level}`                                                                                                      |
| **难度分布**        | 有明确的三个level分布                                                                                                                           |
| **质量评估**        | 基于wikipedia的人工标注+每个答案有明确的supporting facts+难度分层                                                                                          |
| **相关验证**        | Search-R1 论文使用 HotpotQA 作为搜索+推理训练数据；Flow-GRPO 用它作为评估 benchmark（2Wiki, Musique）与 HotpotQA 的 bridge 机制类似；Flow-GRPO 使用 HotpotQA作为benchmark |
